# Learning Workflows

Use this file for beginner explanations, course support, reading roadmaps,
glossaries, or learning plans.

## Stage Routing

| Stage | Goal | Best output |
|---|---|---|
| Beginner | Build vocabulary and avoid concept confusion | Plain explanation, glossary, examples, first readings |
| Coursework | Understand a topic enough to write or present | Concept map, short outline, key sources, discussion questions |
| Literature entry | Start reading papers efficiently | Reading queue, annotation template, key debates |
| Research training | Learn methods, datasets, and evaluation | Dataset/method table, reproducibility plan, small task |
| Proposal/project design | Turn gaps into feasible work | Questions, hypotheses, methods, risks, first experiment |
| Frontier tracking | Keep up with current research | Radar report, ranked candidates, trend notes |

## Beginner Explanation Pattern

Use:

1. One-sentence definition.
2. Why it matters.
3. What it is not.
4. One concrete example.
5. Related terms.
6. First three things to read or search.

Avoid jargon piles. Keep English technical terms visible for later searching.

## Learning Roadmap Pattern

Build roadmaps in levels:

- Level 0: vocabulary and field map.
- Level 1: core wildfire science and fire ecology.
- Level 2: human safety, evacuation, WUI, and refuge planning.
- Level 3: fire refugia, burn severity, recovery, and biodiversity.
- Level 4: remote sensing, GIS, datasets, and uncertainty.
- Level 5: research design, critical reading, and project formulation.

For each level include:

- learning goal
- key concepts
- search terms
- 2-4 reading targets or source types
- small practice task
- common misunderstanding

## Reading Queue Pattern

Rank readings by:

- entry value: clear definitions and field overview
- anchor value: widely cited or methodologically central
- update value: recent evidence or frontier direction
- method value: useful data, code, or reproducible workflow
- user fit: geography, stage, and output goal

Return a table with:

`Priority | Source | Why read | What to extract | Difficulty | Next action`

## Assignment Support Pattern

For coursework, produce:

- thesis or guiding question
- concept map
- 3-5 evidence blocks
- examples or case studies
- limitations and uncertainties
- short bibliography or search plan

Do not overbuild a full research proposal unless the user asks for one.
